## Возможности

ccemaphore — небольшой плавающий светофор, который держится поверх всех окон (в том числе в
полноэкранном режиме). Он сводит состояние всех ваших чатов Claude Code — из всех окон — к одному
цвету, чтобы не переключаться между окнами только ради проверки статуса.

- **Наведите курсор на светофор** — откроется панель со всеми активными сессиями, сгруппированными по
  состоянию **ЖДУТ → РАБОТАЮТ → ГОТОВО**: проект · git-ветка · заголовок чата · % контекста · текущая
  команда. Здесь же — **Обновить**, **История**, **Настройки** и **Выход**.
- **Перетащите** светофор в удобное место; кнопка **«Закрепить»** зафиксирует его.
- **Чат просит разрешение?** Светофор тут же превращается в **ленту** с кнопками **Разрешить один раз /
  Разрешить всё в этом чате / Запретить** и самой командой — можно ответить, не открывая чат.
- **Чат закончил работу?** Рядом со светофором появляется зелёное уведомление «готово»; по кнопке
  **Перейти в чат** откроется нужная вкладка Cursor (или терминал).
- **Идёт сжатие контекста (compacting)?** На жёлтой лампе появляется значок сжатия — значит, занят, а не
  завис.

Всё происходит прямо у светофора — никаких всплывающих уведомлений в углу экрана, которые нужно ловить.

Статистика токенов и расходов по каждой сессии и история по дням берутся из
[`ccusage`](https://github.com/ryoppippi/ccusage), если он установлен.

## Приватность

ccemaphore **читает только локальные файлы** и **ничего не отправляет наружу**. Никакой телеметрии. В
`~/.claude/projects` (транскрипты) он заглядывает лишь ради статуса — не разбирает, не хранит и не
передаёт содержимое переписки. Папка `~/.claude` скрытая (не относится к защищённым TCC-каталогам),
поэтому macOS не спрашивает доступ к ней.

## Как это работает

У ccemaphore два режима, которые дополняют друг друга.

### Режим A — наблюдение за файлами (всегда включён, настройка не нужна)

Через FSEvents он наблюдает за `~/.claude/projects/**/*.jsonl` (транскрипты, которые Claude Code пишет
в реальном времени) и определяет состояние каждой сессии по концу её файла:

- **работает** — последняя запись свежая и относится к незавершённому ходу (`tool_use` ассистента,
  незакрытый стрим, только что пришедший `tool_result`, новый промпт или повтор после `api_error`).
- **готово** — ход завершился штатно (`stop_reason: end_turn`).
- **ждёт** — по возможности: активности давно не было, но сессия ещё жива, а файл обрывается на непарном
  `tool_use`.

Транскрипты суб-агентов приписываются к родительской сессии — пока работает суб-агент, родитель тоже
считается работающим. Активность определяется по последнему реальному `timestamp`, **а не** по mtime
файла: правки метаданных (заголовок, последний промпт) сдвигают mtime уже после того, как чат затих.

### Режим B — хуки (по желанию, в один клик)

Включите **«Точное определение статуса»** в Настройках, и ccemaphore установит хуки Claude Code для:

- **точного определения `done` / `waiting`** — надёжнее, чем чтение конца транскрипта;
- **интерактивной ленты разрешений** (та самая панель из трёх кнопок Разрешить / Всё / Запретить выше);
- **доверенных команд** — списка команд, которые одобряются автоматически, чтобы лента для них не
  появлялась (совпадение по подстроке команды).

Хуки добавляются в `~/.claude/settings.json` аккуратным слиянием (повторная установка ничего не ломает)
и полностью обратимы.

## Установка

1. [**Скачайте последний `.dmg`**](https://github.com/hakkazuu/ccemaphore/releases/latest) — он подписан
   и нотаризован, поэтому Gatekeeper открывает его без предупреждений.
2. Откройте DMG и перетащите **ccemaphore** в **Applications**.
3. Запустите. ccemaphore работает в фоне, без меню и без иконки в Dock — только плавающий светофор.

### Настройка при первом запуске (по желанию, но рекомендуется)

- **Разрешите Универсальный доступ (Accessibility)** (Системные настройки ▸ Конфиденциальность и
  безопасность ▸ Универсальный доступ). Он нужен только для двух вещей: открыть нужное окно/вкладку
  Cursor по кнопке **Перейти в чат** и не показывать уведомление для чата, который у вас и так открыт.
- **Включите «Точное определение статуса»** в Настройках — это активирует режим B (хуки + лента
  разрешений).
- **Для статистики токенов и расходов** нужен Node или Bun в `PATH` — ccemaphore запускает
  [`ccusage`](https://github.com/ryoppippi/ccusage) через `bunx`/`npx`. Без него история всё равно
  работает, просто без цифр по токенам.
- **Если при клике по уведомлению у светофора перемешиваются рабочие столы** — это делает macOS, а не
  ccemaphore: настройка «Менять расположение рабочих пространств Spaces в зависимости от недавнего
  использования» (Системные настройки ▸ Рабочий стол и Dock ▸ Mission Control). Выключите её или выполните
  `defaults write com.apple.dock mru-spaces -bool false && killall Dock` — порядок столов перестанет
  меняться, а переход к нужному окну продолжит работать.

## Участие в разработке и сборка из исходников

Нужны **macOS 13+** и **Xcode 26**. Проект хранится как готовый Xcode-проект (не SwiftPM), так что
`swift build` не сработает — открывайте в Xcode:

```sh
open ccemaphore.xcodeproj   # затем ⌘R — плавающий светофор, без иконки в Dock
```

Собрать локальный (неподписанный) `.app` для быстрой проверки:

```sh
Scripts/package_app.sh    # → build/ccemaphore.app  (Release-сборка через xcodebuild)
```

App Sandbox выключен (иначе не прочитать `~/.claude`), поэтому распространение — мимо Mac App Store.
Issue и pull request приветствуются. Подписанные релизы собирает мейнтейнер через GitHub Actions —
см. [`docs/RELEASING.md`](docs/RELEASING.md).
